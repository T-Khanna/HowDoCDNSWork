Introduction

Content Delivery (or Distribution) Networks, hereafter referred to as CDNs, are interconnected systems of cache servers that can deliver content to users based on their physical, geographical location on Earth1. A cache server is essentially a server that intelligently saves copies web pages, documents, videos, and other media locally2 such that the data can be accessed from closer proximity and more quickly, whilst also reducing bandwidth, server load and ‘lag’3.

This is because requests to a remote server are not needed as much as usual; the files stored on the cache server are updated infrequently and so retrieving the information is not as expensive or time consuming as asking the remote server for the file every time5.

When the first request for the file foo.png is sent, the request is first sent through the surrogate server. However, the surrogate server currently does not contain the file foo.png, so the request is pushed to the origin server. The file foo.png is retrieved and sent back to the user, storing the data for the file in the surrogate server in the process. Due to the caching that occurs, the second and subsequent requests for the same file will stop and retrieve the cached image from the surrogate server.
