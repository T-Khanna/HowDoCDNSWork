Introduction

Content Delivery (or Distribution) Networks, hereafter referred to as CDNs, are interconnected systems of cache servers that can deliver content to users based on their physical, geographical location on Earth1. A cache server is essentially a server that intelligently saves copies web pages, documents, videos, and other media locally2 such that the data can be accessed from closer proximity and more quickly, whilst also reducing bandwidth, server load and ‘lag’3.

This is because requests to a remote server are not needed as much as usual; the files stored on the cache server are updated infrequently and so retrieving the information is not as expensive or time consuming as asking the remote server for the file every time4.

There are many ways in which CDN manage traffic and ensure that users can get the data which is closest to them, and in the fastest time possible. CDN management software can dynamically calculate the closest and fastest servers to a user, and then choose that one to serve the content. The added benefit of this is that there is a lower probability that data can get corrupted and lost on the journey to the user. This is called ‘packet loss’ - a packet is a small collection of data and is how information is transmitted across the world wide web. By reducing distance, the data has less servers to travel between, and so we can be sure that there is less chance of any data going astray1. How this is determined is covered in more detail here.

It is safe to say that without CDNs, the web as we know it today could not exist; Content could not be effectively distributed across the world and load times would be significantly impacted. Additionally, popular web pages receiving lots of concurrent traffic would not be able to cope under the heavy load.
